# ATTLA_T (초장사정 자동선회잠금장치) 프로젝트 명세서

본 문서는 초장사정 자동선회잠금장치(ATTLA_T)의 TMS320F28388D 듀얼 코어 기반 펌웨어 및 시스템의 전체 아키텍처와 세부 기능 스펙을 정리한 문서입니다. 시스템 개발 및 유지보수를 위해 구조화되어 있습니다.

---

## 1. 시스템 개요 및 하드웨어 기본 설정

| 구분 | 상세 내용 |
| :--- | :--- |
| **타겟 장비** | 초장사정 자동선회잠금장치 (ATTLA_T) |
| **메인 MCU** | **TMS320F28388DPTPS** |
| **아키텍처** | **CPU1 단독 구동 + CLA 보조** (CM 코어 및 CPU2 미사용, IPC 로직 제거) |
| **CPU1 역할** | 모터 제어(FOC, PWM), W6100 이더넷 제어(SPI), 전체 시스템 상태 머신 및 운용 제어 |
| **CLA 역할** | 아날로그 센싱(ADC) 데이터 고속 필터링 보조 연산 |

### 1.1 주요 동작 주파수 (Clock Specs)
| 항목 | 주파수 | 비고 |
| :--- | :--- | :--- |
| **CPU1 (C28x) Core** | 200 MHz | 메인 시스템 클럭 |
| **시스템 오실레이터** | 25 MHz | SIT2024BM-S2-33E-25.000000 사용 |
| **이더넷 크리스탈** | 25 MHz | ECS-250-8-37Q-RES-TR 사용 |
| **모터/스위칭 (PWM)** | 10 kHz | 100us 주기 (시스템 운용 Heartbeat 역할) |
| **DC/DC 컨버터** | 600 kHz / 330 kHz | DCM2322T50T2660T60 (600kHz), MGDD-21-N-C (330kHz) |

### 1.2 전원 공급망 (Power Supply)
- **전원 입력 (EMI & 보호)**: PLM74800DRRRQ1, LM5069MM-2 적용 (서지 억제 100V, 스파이크 억제 ±250V, 역전압 보호 -65V). TVS 다이오드(5.0SMDJ120A, 5.0SMDJ33CA)로 MIL-STD-1275E 규격 충족.
- **모터 구동 전원**: 18 ~ 32V 인가 (정격 24V), 최대 사용 전류 12.0A.
- **제어 전원 (3.3V/1.2V)**: 5V 외부 입력 ➡️ TPS70445PWP (고정밀 레귤레이터).
- **통신 전원 (3.3V)**: 체계 통신 안정성을 위해 ADP7158ARDZ-3.3-R7 적용 (독립 분리).

## 2. 모터 구동 및 제어부 (Motor & Drive)

### 2.1 모터 및 기구부 스펙
- **적용 모터**: **maxon EC 60** (24V, 200W)
  - 정격: 24V, 3240 rpm / 정지 토크: 2510 mNm, 정지 전류: 83.2 A
  - 최대 연속 토크: 540 mNm, 최대 연속 전류: 9.34 A
- **기구 구동부**: 기어비 44:1 (모터 44회전 = 기구 1행정)
- **제어 목표**: 위치 오차 ±1° 이내, 동작 제한 속도 1.5초 이내 (24V 인가 기준, 18V 인가 시 초과 허용).

### 2.2 모터 드라이버 (DRV8343)
- **적용 칩셋**: 스마트 게이트 드라이버 **DRV8343SPHPRQ1**
- **인버터 구성**: N-Channel MOSFET (IPTC007N06NM5ATMA1) 6개, 최대 전류 10.0 A. MAX14130/4131 절연 칩 적용.
- **제어 인터페이스 (SPI-B)**:
  - **핀맵**: `CLK` (GPIO 58), `nCS` (GPIO 59), `SIMO` (GPIO 60), `SOMI` (GPIO 61)
- **전용 제어 핀**:
  - `DRV_ENABLE` (GPIO 2, Active High)
  - `DRV_DIR` (GPIO 3, Active High)
  - `DRV_nFAULT` (GPIO 10, 입력, Active Low)

### 2.3 브레이크 (Brake) 제어 회로
- **동작 전류**: 최대 1.0 A
- **제어 핀**: `DSP_BRAKE` (GPIO 35, Active High)
- **회로 구성**: DSP(3.3V) ➡️ NPN TR(MMBT489LT1G) ➡️ TLP293(광절연) ➡️ P-Ch MOSFET(SPD15P10PLGBTMA1) ➡️ 24V 브레이크 구동. 역기전력 방어용 쇼트키 다이오드(V8PAM10HM3/I) 탑재.
- **모니터링**: 브레이크용 TMCS1126 개별 전류 센서를 통해 동작 및 고장 상태 진단.

## 3. 정밀 위치 및 상태 센싱 (Sensing)

### 3.1 모터 위치 검출 (Encoder & Hall)
- **홀 센서**: `HALL_A` (GPIO 11), `HALL_B` (GPIO 12), `HALL_C` (GPIO 13) - SN74HCS126PWR 버퍼 경유.
- **디지털 앱솔루트 엔코더 (RLS AksIM-2)**:
  - **분해능**: 18-bit (싱글턴) + 16-bit (멀티턴 카운터) / 오차: ±0.004° ~ ±0.020°
  - **통신 방식**: **단방향 SPI-C 기반 SSI 통신** (BiSS-C 로직 완전 제거). SN65HVD30MDREP 차동 변환기 탑재.
  - **사용 핀**: `SOMI` (GPIO 51), `CLK` (GPIO 52)
  - **통신 타이밍 규칙**:
    - **초기 대기**: 전원 인가 후 첫 데이터 준비까지 100~200ms 대기.
    - **통신 지연 보상**: 엔코더 응답 지연(170ns) 및 케이블 전파 지연(14ns/m) 발생. 마스터(DSP) 측에서 샘플링 지연 보상 필수. (타임아웃 13.5us)
  - **데이터 프레임 (총 57-bit)**:
    - ACK(13) ➡️ Start(1) ➡️ CDS(1) ➡️ 멀티턴(16) ➡️ 싱글턴(18) ➡️ 에러(1) ➡️ 경고(1) ➡️ CRC(6) (Big Endian 구조)
    - **CRC 규칙**: 0x43 다항식 적용 (위치+에러+경고 총 36비트 대상), 값은 반전(Inverted)되어 전송됨.
  - **소프트웨어 처리 규칙**:
    - **통합 관리**: 멀티턴과 싱글턴을 쪼개지 않고 34비트 변수(`uint64_t`)로 통합. 비트마스크(`0x3FFFFFFFF`)를 사용하여 롤오버/예외 처리를 단순 뺄셈 1회로 간소화.
    - **소프트웨어 제로셋**: 제로셋 명령 시 현재 위치를 FRAM에 오프셋으로 저장하여 뺄셈(-) 연산으로 영점 적용.

### 3.2 아날로그 센서 (ADC)
- **ADC 기본 설정**:
  - **Reference**: 3.0V (REF4132B30DBVRQ1)
  - **Trigger**: EPWM1 SOCA (10kHz, 100us) 독립 동작
  - **입력 보호**: 전 채널 BAT54S 기반 0~3.3V 클램핑 회로 적용.
- **채널 매핑 및 변환 스펙**:

| 센서 / 감시 항목 | 채널 할당 | 스펙 및 특징 |
| :--- | :--- | :--- |
| **모터 전류** | ADCA SOC2 (A2) | TMCS1126 (100mV/A). -24A ~ +24A 구간을 **0.011V ~ 2.982V** 로 변환 (영점 1.5V) |
| **브레이크 전류** | ADCA SOC3 (A3) | TMCS1126. -2.4A ~ +2.4A 구간 변환 (영점 1.5V) |
| **입력 28V 전압** | ADCA SOC4 (A4) | 50V 입력 시 약 **2.964V** 로 변환 |
| **시스템 5V 전압** | ADCA SOC5 (A5) | 5V 입력 시 약 **2.500V** 로 변환 |
| **Reference 전압** | ADCB SOC1 (B1) | REF3040 기반 분배 전압(2.048V) 측정 (ADC 신뢰도 자체 감시용) |
| **보드 온도** | ADCB SOC3 (B3) | MAX6605 내부 온도 센서. -55℃ ~ +125℃를 **0V ~ 2.142V** 로 변환 |

## 4. 체계 통신 및 외부 인터페이스 (Comms & I/O)

### 4.1 체계 통신 (W6100 이더넷)
- 하드웨어 TCP/IP 스택 기반 칩셋(W6100) 적용 (HX1188NL 절연).
- **인터페이스 (SPI-A)**: `SIMO`(16), `SOMI`(17), `CLK`(18), `nCS`(19), `INTn`(20), `RSTn`(21)
- **통신 프로토콜**: UDP, Port 5001, 18바이트 페이로드.
- **수신 동작 구조**: 폴링 방식 폐기. `INTn`(GPIO 20) 핀을 **XINT1 (외부 인터럽트)** 로 수신하여, 패킷 수신 시 하드웨어 인터럽트 단위로 즉각 응답.

### 4.2 디버깅 및 장거리 통신
- **easyDSP**: SCI-A (GPIO 28 `RX`, GPIO 29 `TX`), 115200bps. `PonRST` 신호를 통해 전원/DSP 리셋 동기화.
- **RS-422 통신**: ISOW7843DWE 절연형 트랜시버 탑재. 장거리 차동 신호 송수신 지원.

### 4.3 디지털 입출력 (Digital I/O)
- 입력단 TLP293 포토커플러 절연 회로 적용. 전압 강하(13V 이하) 시 즉각 Fault 인식.

| 구분 | 신호명 (핀) | 핀 방향 및 특징 |
| :--- | :--- | :--- |
| **입력 (센서)** | `nLIMIT1_NO` (36), `nLIMIT1_NC` (37) | 리미트 스위치 1번 (Active Low) |
| **입력 (센서)** | `nLIMIT2_NO` (38), `nLIMIT2_NC` (39) | 리미트 스위치 2번 (Active Low) |
| **입력 (감시)** | `PM_n24V` (40) | 24V 주 전원 감시 (Active Low) |
| **입력 (감시)** | `CABLE_LOOP` (46) | 외부 케이블 연결 체결 감시 (Active Low) |
| **출력 (상태)** | `DSP_LED_nNORMAL` (31) | 외부 시스템 Normal 상태 표시 (Low ➡️ ON) |
| **출력 (상태)** | `DSP_LED_nFAULT` (32) | 외부 시스템 Fault 상태 표시 (Low ➡️ ON) |
| **출력 (LED)** | `LED_G` (30) | 보드 내부 DSP 동작 상태 점멸용 |

### 4.4 비휘발성 메모리 (FRAM)
- **적용 칩셋**: CY15B256Q-SXE
- **인터페이스 (SPI-D)**: `SIMO`(91), `SOMI`(92), `CLK`(93), `CS`(94) / 1MHz 속도.
- **특징**: 데이터 로깅 및 오프셋 저장용. Instant Write 특성을 살려 기존 10ms 블로킹 딜레이를 제거해 지연 없이 운용.

## 5. 구현 현황 및 개발 마일스톤 (Implementation Status)

### ✅ 현재 구현 완료 (Implemented)
- **100us 제어 파이프라인 확립**: `System_Initialization`의 `InitialPeripherals()` 내부에서 주변장치(FRAM, 엔코더)까지 모두 일괄 초기화하도록 캡슐화되었으며, 메인에서는 오프셋➡️PBIT 블로킹 대기만 수행합니다. 이후 EPWM1(100us, 10kHz) 타이머가 중앙 시스템 제어 모듈(CSU)을 일괄 호출하는 Heartbeat 아키텍처가 동작합니다.
- **모듈 통합 리팩토링 (Refactoring)**: 구조적 통일성을 위해 하드웨어 소켓을 직접 제어하던 W6100 로직을 HAL 계층인 `hal_Ethernet` 단일 모듈로 통합하고, EPWM 기반 100us 시스템 타이머 로직을 기존 `hal_Epwm` 모듈로 병합하여 파일 응집도를 극대화했습니다.
- **멀티 코어 의존성 제거**: CM 코어 및 IPC 로직 완전 제거 (CPU1 단독 구동).
- **W6100 (외부 인터럽트 기반 통신망)**: 1ms 폴링을 폐기하고 GPIO 20(`INTn`) 기반 XINT1 외부 인터럽트를 구성하여 즉각적인 패킷 송수신 연동 완료 (`Ethernet_Init`에 일괄 캡슐화).
- **ADC (10kHz 독립 동작)**: EPWM1 트리거를 통해 백그라운드에서 센서 스케일링 및 지수 이동 평균(EMA) 필터링 독립 수행.
- **엔코더 통신**: 복잡한 BiSS-C 로직을 제거하고, 순수 하드웨어 SPI-C 모듈을 통한 SSI 수신 및 데이터 파싱 로직 구현 완료.
- **그 외 주변장치**: DRV8343 모터 드라이버 진단 통신, FRAM 제어(10ms 지연 제거), easyDSP 통신, 전원/리미트 스위치 GPIO 매핑 등 하드웨어 제어망 구축 완료.

### ⏳ 미구현 / 보류 (Pending / On-Hold)
- **내부 EMAC 이더넷**: 내부 컨트롤러 대신 외부 W6100을 통한 통신을 우선하여 채택.

### 🚀 향후 구현 예정 (To Be Implemented)
- **상태 점검 (BIT) 세부 구현**: 100us 파이프라인 내 PBIT/CBIT 프레임워크는 완성되었으며, 하위 센서/하드웨어별 구체적인 고장 진단 및 오프셋 수식 로직 구현 필요.
- **아날로그 / 이산 신호 통합**: 개별로 수집된 센싱 값을 100us 파이프라인(CSU) 내부로 연계하여 논리적 가공.
- **모터 FOC 제어 및 시퀀스**: 엔코더 위치를 바탕으로 한 PID 위치/속도/전류 제어 로직과 브레이크 해제-구동 시퀀스(State Machine) 구현.
- **체계 통신 패킷 확장**: 상위 체계 제어 명령(Command) 패킷 구조체 맵핑 및 CRC 데이터 검증/동작 반영 로직 추가.
